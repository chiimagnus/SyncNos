# NotionAI Web DOM 备注（WebClipper）

这份文档用于补充 `notionai.md`（DOM 快照）里不易从纯 DOM 一眼看出的结构与踩坑点，帮助维护 WebClipper 的 NotionAI 采集逻辑。

## 1) 结构要点

- 页面常见表现为一个“对话滚动区”（A）里包含很多个“对话相关块”（B），而不是严格的一对一 A(B1,B2)。
- **用户消息（B1）**：通常在元素上带 `data-agent-chat-user-step-id`。
- **AI 输出（B2）**：常见为一组块级内容，内部会出现多个 `div[data-block-id]`（Notion block）。
- “Thought”等折叠区块、工具栏按钮、时间戳、头像等都属于**非消息正文**，采集时应尽量排除或不计入正文。

## 2) 采集边界（避免抓到侧栏/正文）

Notion 的页面结构可能同时包含左侧边栏、页面正文、以及聊天区。若采集 root 选得过宽，容易把侧栏或页面正文也当作聊天内容。

建议原则：

- 优先把 root 限定在“对话滚动区”（通常具备 `overflow-y: auto` 的容器），并且仅在其内部寻找 user step 与 assistant blocks。
- 以“用户 step 节点”为锚点，去匹配其后的 assistant 输出容器（直到下一个 user step 出现），避免把全局 `div[data-block-id]` 误识别为 AI 输出。
